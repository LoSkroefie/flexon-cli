Comprehensive FLEXON Guides

This guide provides detailed information on creating schemas, validating and debugging FLEXON data, and leveraging advanced features of the CLI.

1. Writing Schemas

Schemas are JSON-like structures that define the expected structure and types of FLEXON data. Here’s an example schema:

{
    "type": "object",
    "properties": {
        "name": { "type": "string", "minLength": 3 },
        "age": { "type": "integer", "minimum": 0 },
        "email": { "type": "string", "format": "email" }
    },
    "required": ["name", "age"]
}

Explanation:

  • type: Defines the data type (e.g., "object", "string", "integer").
  • properties: Lists the fields in the object and their validation rules.
  • required: Specifies which fields are mandatory.
  • Constraints: Additional constraints like minLength, maximum, or format.

Save this schema as a JSON file (e.g., schema.json) to use for validation.

2. Validating FLEXON Data

FLEXON CLI can validate data against schemas to ensure correctness. Here’s how:

  1. Prepare a FLEXON file (e.g., input.flexon).
  2. Run the validation command: 
    flexon-cli validate input.flexon schema.json
  3. The CLI will output the validation result. If the data fails validation, detailed error messages will be displayed.

Example Output: 

Validation failed:
  - Property 'name': Length 2 is less than minimum 3.
  - Property 'age': Missing required property.

3. Debugging FLEXON Data

The `inspect` command lets you examine FLEXON binary data in a human-readable format:

  1. Run the inspect command: 
    flexon-cli inspect input.flexon [output.json]
  2. If output.json is provided, the inspected data will be saved to a file; otherwise, it will be displayed in the terminal.
  3. Use this command to troubleshoot issues with FLEXON data during development.

4. Advanced Features

4.1 Compression

FLEXON CLI supports GZIP compression for smaller file sizes. Use the --compress flag during encoding:

flexon-cli encode --compress input.json output.flexon

4.2 Verbose Mode

Enable verbose mode to get detailed logs of the encoding or decoding process:

flexon-cli encode --verbose input.json output.flexon

4.3 Depth Limiting

To prevent stack overflow errors, FLEXON CLI supports depth limiting for deeply nested data. The default limit is 100 levels.

You can adjust the depth limit by modifying the source code or extending the CLI with custom arguments.

5. FLEXON CLI Best Practices

  • Always Validate Data: Use schemas to catch data structure issues early.
  • Inspect Before Debugging: Use the `inspect` command to examine data visually.
  • Optimize File Sizes: Enable GZIP compression for large datasets.
  • Log Everything: Use verbose mode to troubleshoot encoding or decoding errors.

6. Integrating FLEXON into Your Workflow

FLEXON CLI is designed to seamlessly integrate into CI/CD pipelines, automated data processing scripts, and cross-platform applications. Here's a quick integration example for automation:

# Bash Script for Automated Encoding and Validation
#!/bin/bash

# Encode JSON to FLEXON
flexon-cli encode input.json output.flexon

# Validate the FLEXON file
flexon-cli validate output.flexon schema.json

# Inspect the FLEXON file
flexon-cli inspect output.flexon debug_output.json

Use this example to automate your FLEXON workflows and ensure consistent data integrity.
